# 编程规范

## 1. 注释规范

### 1.1 中文注释
- 所有代码注释必须使用中文
- 注释应说明"为什么"而不是"是什么"

### 1.2 函数注释
所有函数必须包含文档字符串（docstring），说明：
- 函数功能
- 参数说明
- 返回值说明

示例：
def calculate_discount(price: float, user_level: str) -> float:
    计算用户折扣后的价格
    
    Args:
        price: 商品原价
        user_level: 用户等级（normal/vip/svip）
    
    Returns:
        折扣后的价格

## 2. 类型注解规范

- 所有函数参数必须标注类型
- 所有函数返回值必须标注类型
- 变量声明建议标注类型（复杂类型必标）

示例：
# 正确写法
user_name: str = "张三"
user_list: list[dict[str, Any]] = []

def get_user_info(user_id: int) -> dict[str, Any]:
    pass

## 3. 默认值规范

- 函数参数默认值只能使用 None
- 在函数体内部判断 None 并赋予实际默认值

正确写法：
def query_users(limit: int = None) -> list[User]:
    if limit is None:
        limit = 100
    return users[:limit]

错误写法：
def query_users(limit: int = 100) -> list[User]:
    return users[:limit]

原因：避免可变默认值陷阱（如 def func(lst=[])，并保持一致性。

## 4. Git 提交规范

### 4.1 分批提交原则
- 按功能点分批提交，禁止将多个不相关的功能混在一起提交
- 每次提交只做一件事

### 4.2 提交消息规范
- 提交消息必须使用中文
- 格式：<类型>: <简短描述>

类型说明：
- feat: 新功能
- fix: 修复bug
- docs: 文档更新
- refactor: 代码重构
- style: 代码格式调整
- chore: 构建/工具配置变更

示例：
git commit -m "feat: 添加用户登录功能"
git commit -m "fix: 修复搜索词排名计算错误"
git commit -m "docs: 更新数据表结构文档"
git commit -m "refactor: 提取公共数据库连接函数"

### 4.3 提交前检查
- 代码能正常运行
- 相关测试通过（如有）
- 符合本规范要求
